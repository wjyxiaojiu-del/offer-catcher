# 「Offer 捕手」学生求职匹配智能体 — 方案说明

## 一、问题诊断

学生求职面临两大核心痛点：

**1. 岗位匹配效率低。** 学生需要在各大招聘平台逐一浏览岗位，手动对比自身背景与 JD 要求，耗时且容易遗漏合适岗位。尤其对于跨专业求职或目标不明确的学生，"海投"成为常态，简历与岗位匹配度普遍偏低。

**2. 简历优化缺乏方向。** 学生投递简历后常常石沉大海，却不知道是哪个环节出了问题——是技能不匹配、学历不够、还是简历关键词未覆盖 JD？缺乏系统性的诊断工具，导致反复投递却无法提升通过率。

## 二、方案设计

Offer 捕手采用 **"解析 → 匹配 → 诊断 → 投递"** 四阶段流水线，并在其上叠加一个 **对话式 ReAct Agent** 统一编排：

| 阶段 | 功能 | 技术实现 |
|------|------|----------|
| 解析 | 上传简历（PDF/DOCX/TXT），提取姓名、技能、学历、经历 | unpdf + mammoth + 正则规则引擎，低置信度时 LLM 兜底 |
| 匹配 | 内置岗位 + 用户自定义 JD，四维度打分 | 同义词匹配 + 编辑距离模糊匹配 + 动态权重 |
| 诊断 | 雷达图可视化 + 分析报告 + 逐项优化建议 | 规则评分 + LLM 深度分析（可降级） |
| 投递 | 模拟批量投递 + BOSS 直聘真实自动投递 | 策略筛选 + Playwright 浏览器自动化 |

**核心页面：**
- 首页：简历上传（拖拽 + 文件 + 粘贴）
- 求职 Agent：自然语言对话，自动完成意图识别→规划→执行→记忆
- 匹配结果页：岗位匹配卡片、CountUp 分数动画、雷达图、AI 分析
- JD 优化页：粘贴任意 JD 做针对性简历优化
- 批量投递页：设置匹配度/城市/薪资/类型策略，一键批量投递（模拟）
- BOSS 直聘页：扫码登录后真实自动打招呼投递
- 投递记录页：投递漏斗、状态管理、手动/自动分类

## 三、AI 架构与工具选型

**核心设计哲学：规则优先、LLM 兜底（Rule-first, LLM-fallback）。**

系统并非"纯规则"也非"纯 LLM"，而是双层混合：高频、可解释的路径走规则引擎（0 延迟、0 成本）；歧义、复杂、需要语义理解的场景由 LLM（小米 MiMo）增强。**任一 LLM 调用失败都有规则降级**，保证应用在无 API Key 或 LLM 不可用时仍完整可用。

| 维度 | 选择 | 理由 |
|------|------|------|
| 简历解析 | 正则规则 + LLM 兜底 | 规则覆盖常见中文简历格式；格式异常时 LLM 提取结构化信息 |
| 技能匹配 | 同义词表 + 编辑距离 | 解决 "JS/JavaScript/ES6" 同义词；Levenshtein 容忍拼写错误 |
| 意图识别 | 规则优先 + LLM 兜底 | 常见意图正则秒级命中；模糊表达交给 LLM |
| 任务规划 | LLM 分解 + 规则回退 | LLM 动态编排任务图；失败回退预定义规则模板 |
| 报告生成 | 规则模板 + LLM 深度分析 | 规则给出确定性评分；LLM 补充自然语言洞察 |
| 真实投递 | Playwright 浏览器自动化 | 真实驱动 BOSS 直聘，扫码登录 + 自动打招呼 |

**为什么是混合架构而非二选一：**
1. **可用性兜底**：LLM 失败、超时、无 Key 时自动降级规则引擎，应用永不"罢工"
2. **成本与延迟可控**：高频路径走规则，零 API 调用；仅复杂场景消耗 token
3. **可解释性**：规则路径每个分数都有明确计算逻辑，便于展示算法原理
4. **渐进增强**：LLM 是增益而非依赖，部署门槛低（Vercel 免费层 + SQLite 即可跑）

## 四、ReAct Agent 编排

`/agent` 的每轮对话经过完整 ReAct 流水线（`src/lib/agent/orchestrator.ts`）：

```
用户输入
  → 意图识别（规则优先 / LLM 兜底，3s 超时）
  → 任务规划（LLM 分解任务图 / 规则回退）
  → 任务执行（按依赖拓扑并发：解析→匹配→AI分析→生成报告）
  → 响应生成（LLM / 模板兜底）
  → 记忆持久化（会话短期记忆 + 用户长期偏好 → Prisma）
```

每个 LLM 调用都包裹超时与降级，任务执行支持依赖编排与并发，失败任务不阻塞整体回复。

## 五、关键配置

- **技能同义词库**：覆盖 50+ 组同义词（如 React/ReactJS/React.js），解决简历与 JD 用词不一致
- **动态权重策略**：AI/研究类岗位提高学历权重，应届生岗位提高技能权重
- **模糊匹配阈值**：编辑距离按词长动态调整（>4 字符阈值 1，否则 2），并带长度护栏防止 Java↔JavaScript 误匹配
- **投递策略引擎**：匹配度阈值、城市白名单、薪资下限、公司黑名单、岗位类型五维筛选
- **LLM 超时**：单次调用 12s 上限（`MIMO_TIMEOUT_MS` 可调），超时即降级

## 六、安全与工程化

| 项 | 实现 |
|---|---|
| 鉴权 | 所有数据/AI/自动化 API 经令牌校验（常量时间比较防时序攻击） |
| 限流 | general/ai/boss 三档按 IP 分级 |
| Prompt 注入防护 | 用户内容进入 LLM 前剥离结构标签 + 防护指令 |
| 浏览器自动化门禁 | 令牌 + `BOSS_AUTOMATION_ENABLED=true` 双重开关 |
| 数据持久化 | Prisma + SQLite 单例连接，Agent 记忆模型 |
| 降级链 | 解析/匹配/分析/规划/响应每一环都有规则兜底 |
