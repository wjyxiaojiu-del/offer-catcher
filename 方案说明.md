# 「Offer 捕手」学生求职匹配智能体 — 方案说明

## 一、问题诊断

学生求职面临两大核心痛点：

**1. 岗位匹配效率低。** 学生需要在各大招聘平台逐一浏览岗位，手动对比自身背景与 JD 要求，耗时且容易遗漏合适岗位。尤其对于跨专业求职或目标不明确的学生，"海投"成为常态，简历与岗位匹配度普遍偏低。

**2. 简历优化缺乏方向。** 学生投递简历后常常石沉大海，却不知道是哪个环节出了问题——是技能不匹配、学历不够、还是简历关键词未覆盖 JD？缺乏系统性的诊断工具，导致反复投递却无法提升通过率。

## 二、方案设计

Offer 捕手采用 **"解析 → 匹配 → 诊断 → 投递"** 四阶段流水线架构：

| 阶段 | 功能 | 技术实现 |
|------|------|----------|
| 解析 | 上传简历（PDF/DOCX/TXT），自动提取姓名、技能、学历、经历 | pdf-parse + mammoth + 正则规则引擎 |
| 匹配 | 25 个内置岗位 + 用户自定义 JD，四维度打分 | 技能同义词匹配 + 编辑距离模糊匹配 + 权重动态调整 |
| 诊断 | 雷达图可视化 + AI 分析报告 + 逐项优化建议 | 多维度评分算法 + 模板化报告生成 |
| 投递 | 单个投递 + 一键批量投递（可设置策略） | 条件筛选 + 模拟投递 + 状态追踪 |

**核心页面：**
- 首页：简历上传（支持拖拽 + 文件 + 粘贴）
- 匹配结果页：25 岗位匹配卡片、CountUp 分数动画、雷达图、AI 分析
- JD 优化页：粘贴任意 JD 做针对性简历优化分析
- 自动投递页：设置匹配度/城市/薪资/类型等策略，一键批量投递
- 投递记录页：投递漏斗、状态管理、手动/自动分类

## 三、AI 工具选型理由

**当前方案：规则引擎 + 语义增强（无外部 API 依赖）**

选型考虑如下：

| 维度 | 选择 | 理由 |
|------|------|------|
| 简历解析 | pdf-parse / mammoth + 正则 | 轻量、零依赖、部署成本低，对学生简历格式（中文正则）覆盖率高 |
| 技能匹配 | 同义词表 + 编辑距离 | 解决"JS/JavaScript/ES6"等同义词问题，Levenshtein 距离容忍拼写错误 |
| 语义理解 | 关键词覆盖率 + JD 结构化提取 | 从 JD 中自动提取技能、学历、经验要求，无需调用 LLM |
| 报告生成 | 模板化规则 + 动态权重 | 根据岗位类型（AI/大厂/应届）动态调整评分权重，输出差异化建议 |

**为什么选择规则引擎而非直接调用 LLM API：**
1. **零成本部署：** 无需 API Key，适合课程作业场景，Vercel 免费层即可运行
2. **响应速度：** 匹配计算 < 10ms，无需等待 API 调用（通常 2-5s）
3. **可解释性：** 每个分数都有明确的计算逻辑，便于向评委展示算法原理
4. **数据安全：** 简历数据不离开服务器，保护学生隐私

**扩展路径：** 架构已预留 API 接口层（`/api/match`、`/api/jd-optimize`），可随时接入 Claude API 或其他 LLM，将规则引擎替换为语义匹配模型。

## 四、关键配置

- **技能同义词库：** 覆盖 50+ 组同义词（如 React/ReactJS/React.js），解决简历与 JD 用词不一致问题
- **动态权重策略：** AI/研究类岗位提高学历权重（0.25），应届生岗位提高技能权重（0.45）
- **模糊匹配阈值：** 编辑距离 ≤ 2 且长度 > 3 时视为匹配，平衡准确率和召回率
- **投递策略引擎：** 支持匹配度阈值、城市白名单、薪资下限、公司黑名单、岗位类型五维筛选
